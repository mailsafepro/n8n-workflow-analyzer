# n8n Workflow Cost & Risk Analyzer — Demo Pack

## What this is

A **local, offline** static analysis tool for n8n workflow exports.

It reads standard JSON exports from n8n and produces a structured audit pack:

- **Cost risk detection** — AI/LLM nodes, frequent polling, loops with HTTP, many external API calls
- **Operational risk detection** — missing error handling, webhook chaining, DB in loops
- **Complexity detection** — large workflows, high fan-out nodes
- **Node inventory** — what node types are used, how many, which are triggers
- **Connection analysis** — edges, isolated nodes, fan-out patterns
- **Credential reference scan** — which nodes reference credentials (secrets never inspected)

Everything runs on your machine. No data is sent anywhere.

## What this is NOT

- Not an official n8n product or service
- Does not host, store, or execute workflows
- Does not inspect credential values, passwords, tokens, or secrets
- Does not call any external APIs (n8n or otherwise)
- Does not provide runtime metrics or execution logs
- Not a security or compliance audit
- Not a replacement for production monitoring

## Files included

| File | Purpose | Best for |
|------|---------|----------|
| `executive_summary.md` | High-level findings, risk scores, business impact | Stakeholders, management |
| `client_action_plan.md` | Prioritized 7-day remediation plan | Engineering teams |
| `technical_report.md` | Full analysis with all findings per workflow | Technical leads, architects |
| `analysis.json` | Structured data for custom processing | Developers, automation |

## How to read this demo

1. **Start with `executive_summary.md`** — understand the overall risk posture (5 min)
2. **Read `client_action_plan.md`** — see what actions are recommended (5 min)
3. **Dive into `technical_report.md`** — review per-workflow findings (10 min)
4. **Use `analysis.json`** only if you need to integrate with other tools
