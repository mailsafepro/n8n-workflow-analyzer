Automation rarely fails all at once. It gets expensive first.

n8n workflows start small. A webhook here, an HTTP request there. Then someone adds an AI node because the use case calls for it. Then a loop to process items in batches. Then another API integration because the workflow needs data from somewhere else.

Before long you have dozens of workflows and no clear picture of which ones could get expensive, which ones will fail silently, or which credentials are referenced where.

Nobody adds error handling on day one. Nobody sets rate limits until something breaks. Nobody documents which workflow owns which credential.

This tool doesn't fix any of that. But it tells you where to look first.

Privacy

Workflow data should not need to leave your machine.

The analyzer was built to run without network access. No telemetry, no cloud processing, no API calls. You export JSON files from n8n, anonymize anything sensitive, and run the analysis locally. That's it.

This is a cost and operational risk screening. It is not a security or compliance audit.

Security note
— Runs locally / offline — no server-side processing
— No workflow execution — static analysis only
— No calls to n8n API or any external service
— No credential values inspected or stored
— No n8n account access needed
— Anonymized exports only — remove sensitive data before sharing

FAQ

Common questions

Is this affiliated with n8n?
No. Independent tool, not affiliated with or endorsed by n8n GmbH. "n8n" is used to describe what the tool analyzes.
Do I need to give access to my n8n instance?
No. The analyzer reads exported JSON files only. No API keys, no instance access, no login.
Do you see my credentials?
No. The analyzer counts credential references — it never reads, inspects, or stores values, tokens, or secrets. Anonymize sensitive data before sharing.
What files do I provide?
Standard n8n workflow exports in JSON. Use the export function in the editor. Replace API keys, internal URLs, and emails with placeholders before sharing. Send 3–10 production-critical workflows.
Does it estimate exact costs?
No. Cost estimates are approximate, based on configurable assumptions — not your actual bills. The tool identifies patterns that create cost risk. Actual costs depend on trigger frequency, data volume, and API pricing.
Is this a security audit?
No. This is a cost and operational risk screening. It does not inspect credentials, data handling, access controls, or compliance. For security audits, consult a qualified professional.
What do I receive?
Four files: executive summary, client action plan, technical report, and analysis JSON. Each targets a different audience — stakeholders, engineers, or automation tooling.
Can consultants use this for client audits?
Yes. The deliverables are designed to be shareable. You're free to incorporate them into your own consulting reports.
